# 四星讲师海报生成工具开发守则

本项目用于自动生成「懂车学四星讲师」认证海报。它不是一个重新设计海报的项目，而是一个基于既有海报视觉模板，输入讲师信息后自动生成成品海报的工具。

本文件优先服务后续开发者和维护者，同时也作为 AI 开发代理的执行约束。任何开发动作都必须先遵守本文规则，再考虑具体实现方案。

## 项目核心原则

1. 不重新设计海报。
2. 不改变四星讲师海报的主视觉风格。
3. 不随意移动固定视觉元素。
4. 不为了容纳更多文字而自动缩小字号。
5. 所有可调参数尽量放入模板配置文件，而不是藏在代码里。
6. 第一阶段先保证命令行生成效果正确，再考虑图形界面。

## 第一阶段范围

第一阶段只实现「懂车学四星讲师」单模板。

必须支持：

- 单个讲师信息生成海报。
- Excel / CSV 批量导入。
- 批量导出 PNG / JPG。
- 固定模板自动排版。
- 通过配置文件管理坐标、样式、字效和布局参数。
- 人像替换，支持透明 PNG，也允许引入自动抠图流程。

暂不实现：

- 三星讲师模板。
- 陪跑讲师模板。
- 多模板 UI 切换。
- 拖拽式可视化编辑器。
- 在线后台。
- 复杂用户权限。

代码结构需要预留多模板扩展能力，后续可以增加 `three_star`、`coach` 等模板目录。

## 固定内容

以下内容不得被程序重新设计、替换风格或随意重绘：

- 海报尺寸。
- 海报比例。
- 背景。
- 光效。
- Logo。
- 麦克风。
- 道路。
- 汽车。
- 主标题「懂车学 认证讲师」。
- 固定文字「懂车学四星讲师」。
- 整体黑金视觉风格。
- 所有装饰元素。

如果后续提供分层 PNG 素材，程序可以按原视觉关系进行合成，但不能改变这些元素的设计语言、位置关系和层级逻辑。

## 可变内容

程序只允许替换以下内容：

- 讲师姓名。
- 项目经历。
- 底部讲师介绍。
- 服务品牌文字。
- 讲师人像。

服务品牌属于底部讲师介绍正文的一部分，不需要单独做特殊样式。

## 模板实现方式

第一阶段以既有四星讲师海报作为固定模板基础。

开发时应理解为：

- 固定视觉元素来自模板底图或后续分层素材。
- 可变内容由程序替换到指定位置。
- 程序负责文字排版、人像合成和导出。
- 程序不负责重新创作背景、标题、光效、麦克风、汽车、道路等主视觉元素。

## 配置文件原则

模板配置文件可以理解为“海报尺子和颜料盒”：它告诉程序每个元素放在哪里、画多大、用什么颜色、有没有字效。代码只负责读取这些规则并执行。

坐标、尺寸、字体、字号、颜色、行距、最大行数、渐变、投影、描边、动态布局参数等，优先放入：

```text
templates/four_star/config.json
```

代码中不应散落大量固定坐标和样式数值。若开发早期必须临时写入默认值，需要尽快迁移到配置文件。

配置文件至少应覆盖：

- 海报尺寸。
- 模板路径。
- 字体路径。
- 字体字效配置。
- 项目经历区域坐标、宽度、字号、颜色、行距、最大行数、底部 baseline。
- 姓名坐标、字号、颜色、字间距规则、渐变、投影等字效。
- 固定文字「懂车学四星讲师」的位置和样式。
- 底部介绍框宽度、最小高度、最大高度、内边距、描边样式。
- 底部介绍文字字号、颜色、行距、最大行数。
- 「懂车帝 - 巨懂车」的位置计算规则、字号、颜色。
- 底部动态布局计算区域。
- 人像位置、尺寸、裁切规则、层级。
- 输出格式和导出质量。

## 字体与字效

原海报中的文字存在字体效果时，后续实现必须尽量还原，包括但不限于：

- 渐变。
- 投影。
- 描边。
- 高光。
- 字间距。
- 层叠文字效果。

不能因为实现方便就把有字效的文字简化成普通纯色文字。若某种字效暂时无法实现，需要在代码或文档中明确标记为待完善项，并保留扩展入口。

## 全局文字规则

所有可变文字默认遵守：

- 使用固定字体。
- 使用固定字号。
- 使用固定颜色或固定字效。
- 使用固定行距。
- 自动换行。
- 不自动缩小字号。
- 不随内容拉伸变形。
- 不允许超出规定区域。
- 超出区域的内容直接截断或不显示。

任何区域都不能通过缩小字号来强行容纳更多内容。

## 项目经历排版规则

项目经历区域采用“底部锚定，向上生长”的排版方式。

必须满足：

- 最后一行位置固定。
- 最后一行始终落在配置指定的 baseline 上。
- 内容较少时，文本块从底部向上减少。
- 内容较多时，文本块从底部向上增长。
- 固定字号、固定行距、固定颜色或固定字效。
- 自动换行。
- 不改变区域宽度。
- 不自动缩小字号。
- 超出最大允许行数后，顶部多余内容截断或不显示。

开发和验收时要优先检查这一点，因为它是四星讲师模板最关键的排版规则之一。

## 讲师姓名规则

讲师姓名位置固定，不能因为姓名长度变化而上下移动。

必须满足：

- 姓名基准位置保持原模板位置。
- 姓名不能上下移动。
- 姓名不能随意左右漂移。
- 尽量保持原模板字号、字体和字效。
- 姓名较长时，优先轻微调整字间距。
- 不建议缩小字号；默认禁止自动缩小字号。
- 姓名右侧竖线与「懂车学四星讲师」的视觉关系保持稳定。
- 「懂车学四星讲师」文字内容不变。

## 底部讲师介绍框规则

底部介绍框是动态高度区域。

必须满足：

- 介绍框宽度固定。
- 介绍框左右位置固定。
- 介绍框只能上下改变高度。
- 不允许左右拉宽。
- 介绍文字使用固定字号。
- 介绍文字使用固定行距。
- 介绍文字使用固定内边距。
- 自动换行。
- 不自动缩小字号。
- 最多显示 6 行。
- 超过 6 行的文字直接截断，不显示。
- 文字不得溢出介绍框。
- 介绍框高度根据实际显示行数变化。

如果介绍文字只有 2 行，介绍框高度按 2 行内容计算；如果有 6 行，介绍框高度按 6 行内容计算。

## 底部动态布局规则

底部区域包含四个垂直对象：

1. 「讲师姓名 + 懂车学四星讲师」
2. 底部讲师介绍框
3. 「懂车帝 - 巨懂车」
4. 海报最底端

其中：

- 第 1 项位置固定。
- 第 2 项高度可变。
- 第 3 项必须根据介绍框高度动态计算位置。
- 第 4 项固定不变。

核心要求：

```text
Gap1 = Gap2 = Gap3
```

含义：

- Gap1：姓名区域底部到介绍框顶部。
- Gap2：介绍框底部到「懂车帝 - 巨懂车」顶部或中心参考线。
- Gap3：「懂车帝 - 巨懂车」底部或中心参考线到海报最底端。

不能把「懂车帝 - 巨懂车」的位置写成固定坐标。它必须根据介绍框高度和底部区域剩余空间动态计算。

## 人像规则

人像可以替换。

第一阶段支持：

- 透明 PNG 人像。
- 非透明图片自动抠图流程。

必须满足：

- 人像位置保持模板原位置。
- 人像大小保持模板原大小。
- 人像裁切方式保持一致。
- 人像层级在背景之上。
- 人像不能遮挡固定主标题。
- 人像与麦克风、道路、汽车的前后层级按原模板保持。

自动抠图属于允许引入的复杂流程，但如果效果不稳定，需要保持可关闭或可替换，不能影响透明 PNG 的稳定生成。

## 数据导入

单个输入至少包含：

- 讲师姓名。
- 项目经历。
- 讲师介绍。
- 人物图片路径。

批量导入支持 Excel / CSV。字段名称可以根据实际实现调整，但需要在 README 和样例文件中清楚说明。

建议字段：

```text
name
project_experience
bio
portrait_path
output_filename
```

项目经历可以支持换行或分号分隔。

## 导出要求

必须支持：

- PNG。
- JPG。
- 单张导出。
- 批量导出到 `output/` 文件夹。

导出必须满足：

- 尺寸与模板完全一致。
- 比例与模板完全一致。
- 图片清晰。
- 不因为压缩导致文字和人像变糊。
- 文件名可以使用讲师姓名自动生成，也可以使用导入数据中的输出文件名。

## 技术方向

推荐使用 Python 实现。

优先考虑：

- Pillow：文字绘制、字效、图片合成。
- OpenCV：图片处理和必要的抠图辅助。
- pandas / openpyxl：读取 Excel。
- CSV 标准库或 pandas：读取 CSV。

第一阶段优先命令行版本。命令行跑通并通过验收后，再考虑 Tkinter、PyQt 或其他图形界面。

## 开发顺序建议

1. 建立项目结构和四星讲师模板配置。
2. 实现配置读取。
3. 实现文字换行、截断和行数限制。
4. 实现项目经历“底部锚定，向上生长”。
5. 实现姓名区域绘制和字间距处理。
6. 实现底部介绍框动态高度。
7. 实现底部 `Gap1 = Gap2 = Gap3` 的动态布局。
8. 实现人像替换和层级合成。
9. 实现单张命令行生成。
10. 实现 Excel / CSV 批量生成。
11. 增加样例数据和验收说明。

## 验收要求

每次关键改动后，必须使用样例数据生成海报并检查视觉结果。

至少检查：

- 固定背景和主视觉没有改变。
- 主标题没有改变。
- Logo 没有改变。
- 麦克风、道路、汽车、光效没有改变。
- 项目经历最后一行位置固定。
- 项目经历向上生长。
- 姓名位置固定。
- 「懂车学四星讲师」内容不变。
- 介绍框宽度固定。
- 介绍框高度根据文字行数变化。
- 介绍文字最多显示 6 行。
- 超出 6 行的文字不显示。
- 所有文字都没有自动缩小字号。
- 「懂车帝 - 巨懂车」根据介绍框高度动态移动。
- 底部区域满足 `Gap1 = Gap2 = Gap3`。
- 人像层级正确。
- 导出的 PNG / JPG 清晰。
- 批量导入能正常生成多张海报。

## 禁止事项

开发过程中禁止：

- 把四星讲师海报重新设计成另一种风格。
- 用代码重绘一套新的背景、光效、汽车、麦克风或道路。
- 为了适配文字长度自动缩小字号。
- 让文字溢出指定区域。
- 把「懂车帝 - 巨懂车」底部位置写死。
- 把大量坐标和样式散落在代码中。
- 一开始就优先开发 GUI 而忽略生成效果。
- 为了实现方便而删除原海报中的字效。

## 面向 AI 开发代理的特别要求

在修改代码前，AI 开发代理必须先确认当前任务是否会影响固定视觉元素、动态排版规则或模板配置结构。

如果任务涉及排版，优先检查：

- 项目经历 baseline。
- 介绍框最大 6 行。
- 底部三段等距。
- 固定字号不缩小。

如果任务涉及素材，优先检查：

- 固定主视觉是否被改动。
- 人像层级是否正确。
- 是否保留透明 PNG 的稳定路径。
- 自动抠图是否可关闭或可替换。

如果任务涉及新增模板，不允许直接污染 `four_star` 的配置和代码，应通过模板目录和配置扩展完成。
